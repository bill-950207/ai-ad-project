현재까지의 작업을 중간 커밋합니다.

사용법: /checkpoint [커밋 메시지]
예시:
  /checkpoint
  /checkpoint WIP: 아바타 생성 API 구현 중
  /checkpoint feat: 제품 목록 조회 API 완성

## 실행 절차

### 1. 변경사항 확인
- `git status`로 변경된 파일 확인
- `git diff`로 변경 내용 확인
- 변경사항이 없으면 "커밋할 변경사항이 없습니다"라고 알리고 종료

### 2. 커밋 메시지 결정
- $ARGUMENTS가 있으면 해당 메시지 사용
- $ARGUMENTS가 없으면 변경 내용을 분석하여 자동 생성
  - 변경 범위가 명확하면: `feat:`, `fix:`, `refactor:` 등 적절한 prefix
  - 작업 중간 상태면: `WIP: 작업 내용 요약`

### 3. 스테이징 및 커밋
- 변경된 파일을 개별적으로 `git add` (민감 파일 제외: .env, credentials 등)
- 새로 생성된 파일이 있으면 목록을 보여주고 포함 여부 확인 없이 관련 파일 모두 추가
- 커밋 실행

### 4. 결과 보고
- 커밋 해시, 메시지, 변경된 파일 수를 간단히 알려줍니다
